## Technical Analysis

**Architecture:**
- SQLite-based persistent message storage
- TCP socket communication on localhost (127.0.0.1)
- MCP server providing natural language tools
- Session-based authentication with optional shared secrets

**Message Storage:**
- Messages queued for offline recipients
- Large file handling for attachments
- Metadata tracking (sender, timestamp, content)
- Cross-session persistence

**Key MCP Tools:**
- `register`: Create instance identity
- `send`: Direct messaging to recipient
- `check`: Retrieve new messages
- `broadcast`: Message all instances
- `share_file`/`share_command`: File/output sharing
